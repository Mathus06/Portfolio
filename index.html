Voici un exemple de ce que l'on a pu réaliser au Club Informatique l'année dernière avec mon professeur. On y retrouve principalement du HTML et surtout du PHP (et SQL) avec lesquels nous avons créé ce Morpion qui est un jeu en local (avec 2 ordinateurs). Nous n'avons pas développé la partie CSS tout simplement du fait que ce n'était pas le but premier du projet.

Github ne permet pas d'ajouter de bases de données, je ne peux donc pas vous laisser prendre plaisir à jouer au morpion entre vous.

Nous deviens également pour les vacances de Noël de cette année, réaliser un site internet permettant de gérer un stockage, pour ce projet j'ai arboré une devanture graphique simple et très peu avancé mais le but était que cela ressemble plus ou moins à un entrepôt (malgré les couleurs que je n'ai pas assorties entres-elles par ailleurs) afin de pouvoir me tester sur mes connaissances en web et également montrer au professeur ce que je savais faire sans pour autant passer trop de temps dessus. Le site est fait de ce fait en HTML, CSS, Javascript et PHP/SQL; Javascript du fait que la partie en bleu n'apparaît pas directement, il faut cliquer sur l'un des boutons du dessus.

Notre professeur de NSI a décidé de nous faire réaliser un projet de fin d'année parmi plusieurs thèmes choisis par celui-ci. Nous sommes une équipes de 3 personnes et nous avons choisi de réaliser un Sudoku en Python avec l'aide de la bibliothèque tkinter pour la partie graphique du code.

Nous venons tout juste de commencer ce projet donc nous n'avons que la création d'une grille vide, de l'implémentation d'une grille de Sudoku (partielle ou complète) (comme vous pouvez le voir sur l'image), et bien évidemment l'ajout graphique.

Nous avons pour premièrement étape de réaliser un Sudoku solvable par l'ordinateur, vous pouvez retrouver notre plan sur ce Figma, puis de réaliser un Sudoku jouable par un utilisateur et dont la correction se fera par l'algorithme de correction réalisé dans notre objectif précédent.

Projets Personnels

Ce projet que j'ai intégré à ma page est testable, il vous suffit de cliquer sur le "Jouer 1000 fois" pour lancer la simulation.

Cette idée de projet fondée en HTML, CSS et Javascript m'est venu de cette vidéo (<- Cliquez pour l'ouvrir) du journal Le Monde sur Tiktok dans laquelle une programmeuse du journal a créé ce simulateur du célèbre jeu à gratter Cash. N'ayant que le visuel j'ai tenté de reproduire au mieu l'aspect graphique et le code fait par cette programmeuse afin d'avoir ce résultat qui se rapproche de ce qu'elle a pu faire. Pour cela j'ai utilité la bibliothèque ChartJS pour le graphique, ce qui m'a permis de comprendre les mécanismes pour en faire.

Durant ce petit projet qui m'a pris un weekend, j'ai rencontré peu de difficultés à le réaliser, mais j'ai dû demander de l'aide pour importer en temps réel les données sur le graphique (ce que je n'avais jamais fait auparavant) et régler un léger problème dans mes statistiques.

Dans le but de développer mes compétences en backend, je travaille sur le PHP/SQL et pour découvrir des fonctionnalités intéressantes et développées, je suis un tutoriel afin de créer une messagerie instantanée simple.

Pour cela, je suis la méthodologie et analyse le code en détail dans le but de comprendre les mécanismes d'échange entre serveur et page web.
